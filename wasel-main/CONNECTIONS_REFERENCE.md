# 🔀 جدول مقارنة الأدوار والاتصالات

## 📊 جدول شامل

| العنصر | الموصل (Driver) | المشرف (Supervisor) | المدير (Admin) |
|------|-----|---------|---------|
| **الدور في النظام** | `delivery_person` | `supervisor` | `admin` |
| **الرابط** | `/DriverPanel` | `/SupervisorPanel` | `/StaffDashboard` |
| **من يملك الوصول؟** | الموصلون فقط | المشرفون فقط | المدير فقط |
| **مرئي في الملاح؟** | ✅ نعم | ✅ نعم | ❌ مخفي |

---

## 🗄️ الجداول المستخدمة (في كل صفحة)

### **DriverPanel**
| الجدول | العملية | الوصف |
|-------|--------|-------|
| `admin_users` | SELECT | التحقق من دوره = delivery_person |
| `order_assignments` | SELECT | جلب الطلبات حيث delivery_person_id == user.id |
| `orders` | SELECT + UPDATE | عرض التفاصيل، تحديث الحالة إلى "delivered" |

**الاستعلام:**
```sql
SELECT 
  oa.*, 
  o.* 
FROM order_assignments oa
JOIN orders o ON oa.order_id = o.id
WHERE oa.delivery_person_id = :user_id
```

---

### **SupervisorPanel**
| الجدول | العملية | الوصف |
|-------|--------|-------|
| `admin_users` | SELECT | التحقق من دوره = supervisor |
| `orders` | SELECT | جلب جميع الطلبات بدون تصفية |
| `order_assignments` | SELECT | (اختياري) لمعرفة الموصل |

**الاستعلام:**
```sql
SELECT * FROM orders
ORDER BY created_at DESC
```

---

### **StaffDashboard**
| الجدول | العملية | الوصف |
|-------|--------|-------|
| `admin_users` | SELECT | جميع الموظفين النشطين |
| `orders` | SELECT + UPDATE | جميع الطلبات، تحديث الحالة |
| `order_assignments` | INSERT + UPDATE | تعيين طلب لموصل |

**الاستعلامات:**
```sql
-- جلب الموظفين
SELECT * FROM admin_users WHERE is_active = true

-- جلب الطلبات
SELECT * FROM orders

-- تعيين طلب
INSERT INTO order_assignments (order_id, delivery_person_id, assigned_by, status)
VALUES (:order_id, :driver_id, :admin_id, 'assigned')
```

## 💾 الدوال الخاصة (Functions)

```
┌─────────────────────────────────────────────────────────┐
│ الدالة 1: handle_new_user()                           │
├─────────────────────────────────────────────────────────┤
│ عند: إنشاء مستخدم جديد في Supabase Auth              │
│ تفعل: إضافة سجل تلقائي في جدول admin_users          │
│                                                         │
│ السبب: ربط بيانات الحساب مع دور المستخدم             │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ الدالة 2: update_order_status_on_assignment()        │
├─────────────────────────────────────────────────────────┤
│ عند: تحديث حالة التعيين (assignment)                  │
│ تفعل: تحديث حالة الطلب (order) تلقائياً              │
│                                                         │
│ مثال:                                                  │
│ - الموصل غير الحالة إلى "in_progress"                │
│ - الدالة تحدث حالة الطلب إلى "in_progress"           │
│                                                         │
│ الفائدة: مزامنة الحالات بين الجداول                   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ الدالة 3: calculate_distance(lat1, lon1, lat2, lon2) │
├─────────────────────────────────────────────────────────┤
│ الدخل: موقعين جغرافيين (خيط وعرض)                    │
│ الخرج: المسافة بالكيلومتر                           │
│                                                         │
│ الاستخدام: إيجاد أقرب موصل للطلب                    │
│                                                         │
│ الصيغة: Haversine Distance Formula                    │
│ (حساب المسافة على سطح الكرة الأرضية)                │
└─────────────────────────────────────────────────────────┘
```

## 🔄 Triggers (المشغلات)

```
┌─────────────────────────────────────────────────────────┐
│ Trigger 1: on_auth_user_created                       │
├─────────────────────────────────────────────────────────┤
│ متى: عند إنشاء مستخدم جديد في auth.users            │
│ ماذا: تشغيل دالة handle_new_user()                   │
│                                                         │
│ التسلسل:                                               │
│ 1. المستخدم ينقر "تسجيل دخول" في StaffLogin        │
│ 2. Supabase Auth ينشئ المستخدم                      │
│ 3. Trigger ينشغل تلقائياً                           │
│ 4. الدالة تضيف السجل في admin_users                │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ Trigger 2: on_assignment_status_change              │
├─────────────────────────────────────────────────────────┤
│ متى: عند تحديث status في order_assignments          │
│ ماذا: تشغيل دالة update_order_status_on_assignment() │
│                                                         │
│ التسلسل:                                               │
│ 1. الموصل يضغط "تم التسليم" في DriverPanel         │
│ 2. يحدث status = "delivered" في order_assignments    │
│ 3. Trigger ينشغل                                    │
│ 4. الدالة تحدث status = "delivered" في orders       │
└─────────────────────────────────────────────────────────┘
```

## 📈 مثال كامل: من الدخول إلى التسليم

```
┌──────────────────────────────────────────────────────┐
│ الخطوة 1: الموصل يدخل                              │
│ - يذهب إلى /stafflogin                            │
│ - يختار "موصل"                                     │
│ - يدخل بريده وكلمة مروره                           │
└──────────────────┬───────────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────────┐
│ الخطوة 2: التحقق والتوجيه                          │
│ - Supabase Auth يتحقق من البيانات ✓              │
│ - تحميل البيانات من admin_users ✓               │
│ - التربيز من الدور = delivery_person ✓           │
│ - توجيه إلى /DriverPanel ✓                       │
└──────────────────┬───────────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────────┐
│ الخطوة 3: عرض الطلبات                              │
│ - الاستعلام: SELECT من order_assignments          │
│   WHERE delivery_person_id = user.id              │
│ - ربط مع orders للحصول على التفاصيل              │
│ - عرض الطلبات في القائمة                          │
└──────────────────┬───────────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────────┐
│ الخطوة 4: التسليم                                  │
│ - الموصل يضغط "تم التسليم"                        │
│ - UPDATE order_assignments:                       │
│   status = "delivered"                            │
│ - Trigger ينشغل:                                 │
│   UPDATE orders: status = "delivered"            │
└──────────────────┬───────────────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────────────┐
│ النتيجة: الطلب مكتمل ✓                            │
│ - الموصل: رأى "تم التسليم"                        │
│ - المشرف: يرى الحالة محدثة                        │
│ - المدير: يرى كل شيء محدث                        │
└──────────────────────────────────────────────────────┘
```
