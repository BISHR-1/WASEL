## 💳 اختبارات Payment Security

### ✅ Test 4: منع الدفع بدون طريقة دفع

**الخطوات:**
```
1. أضف عنصر إلى السلة
2. اذهب إلى الدفع
3. اترك "طريقة الدفع" فارغة
4. انقر على "تأكيد الشراء"
5. توقع: رسالة خطأ واضحة
```

**رسالة الخطأ المتوقعة:**
```
"يجب اختيار طريقة دفع"
```

**الكود الذي يتحقق:**
```javascript
// validatePaymentBeforeOrder() checks:
if (!['paypal', 'wallet', 'whatsapp'].includes(orderData.paymentMethod))
  throw new Error('Invalid payment method');
```

---

### ✅ Test 5: منع الدفع برقم سالب

**الخطوات:**
```
1. أضف عنصر للسلة (سعر: 50 USD)
2. اذهب للدفع
3. حاول تعديل السعر إلى -50 USD (عبر DevTools)
4. اضغط Checkout
5. توقع: الرفض وتسجيل نشاط مريب
```

**النتيجة:**
- ✅ PASS: رفض الطلب ورسالة خطأ
- ✅ تسجيل في suspicious_activities_log
- ❌ FAIL: تم الدفع برقم سالب!

---

### ✅ Test 6: منع الدفع برقم فوق الحد الأقصى

**الخطوات:**
```
1. أضف عنصر مشروع
2. عدّل الطلب: "totalAmount": 50000 (بدلاً من 50)
3. اضغط Checkout
4. توقع: الرفض
```

**الحد الأقصى المسموح:** $10,000

**الكود:**
```javascript
// validateOrderAmountSane()
if (amount < 0.01 || amount > 10000)
  throw new Error('Amount outside valid range');
```

## 👝 اختبارات Wallet Security

**الخطوات:**
```

**الكود الآمن:**

```

---

### ✅ Test 8: منع الدفع برصيد أقل من المطلوب

**الخطوات:**
```
1. تحقق من رصيد المحفظة: 20 USD
2. حاول شراء شيء بقيمة: 50 USD
3. اختر محفظة كطريقة دفع
4. توقع: رفض العملية
```

**رسالة الخطأ:**
```
"الرصيد غير كافي. الرصيد الحالي: 20 USD، المبلغ المطلوب: 50 USD"
```

**الكود:**
```javascript
// validateWalletPayment()
if (walletBalance < requiredAmount)
  throw new Error('Insufficient balance');
```

## 🚫 اختبارات Fraud Detection

### ✅ Test 9: منع الطلبات المكررة (30 ثانية)

**الخطوات:**
```
1. أضف عنصر + أكمل الدفع
2. مباشرة (خلال 10 ثوانٍ)، حاول إعادة نفس الطلب
3. اضغط Submit مرة أخرى
4. توقع: الرفض
```

**رسالة الخطأ:**
```
"تم إنشاء طلب مماثل للتو. يرجى الانتظار قبل إعادة المحاولة"
```

**الآلية:**
```javascript
// checkDuplicateOrder() uses hash:
const orderHash = btoa(JSON.stringify({
  items: cart.items,
  total: cart.total,
  recipient: recipient || 'unknown'
}));
```

### ✅ Test 10: منع تعديل الأسعار أثناء الدفع

**الخطوات:**
```
1. أضف عنصر (سعر: 10 USD)
2. في صفحة الدفع، استخدم DevTools
3. غير: "itemPrice": 1 (بدلاً من 10)
4. اضغط Checkout
5. توقع: الرفض
```

**النتيجة:**
- ✅ PASS: رفض الطلب
- ✅ تسجيل النشاط المريب
- ❌ FAIL: تم الدفع برسم مخفض!

---

### ✅ Test 11: منع PayPal Spoofing

**الخطوات:**
```
1. ابدأ دفع PayPal
2. في الـ Response، غير: "status": "FAILED" (بدلاً من COMPLETED)
3. نسخ/لصق الاستجابة المزيفة
4. توقع: الرفض
```

**التحقق:**
```javascript
// validatePayPalCapture()
if (captureData.status !== 'COMPLETED')
  throw new Error('PayPal transaction not completed');
```

## 📊 اختبارات Logging & Audit

### ✅ Test 12: التحقق من تسجيل النشاط المريب

**الخطوات:**
```
1. أكمل اختبار الاحتيال أعلاه
2. اذهب إلى /SupervisorPanel
3. انقر على "نشاط مريب" أو "Suspicious Activities"
4. توقع: رؤية جميع محاولات الاحتيال المسجلة
```

**البيانات المسجلة يجب أن تشمل:**
```json
{
  "user_id": "...",
  "activity_type": "invalid_payment_amount",
  "reason": "Amount outside valid range",
  "attempted_amount": 50000,
  "created_at": "2026-03-11T...",
  "details": { "error": "Amount exceeds $10,000 limit" }
}
```

**الخطوات:**

3. اذهب إلى الملف الشخصي > المحفظة
4. انقر على "سجل العمليات"
5. توقع: رؤية جميع الإضافات والخصومات
```

**البيانات المسجلة:**
```
- إضافة: +50 USD (PayPal) - الرصيد: 0 → 50
